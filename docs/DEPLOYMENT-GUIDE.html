Database Setup

Configure PostgreSQL for the backend

Option A: Local PostgreSQL

  1. Install PostgreSQL from postgresql.org
  2. Open pgAdmin or terminal and create a database:
CREATE DATABASE portfolio_db;

Option B: Neon Cloud PostgreSQL (Recommended)

⭐
Why Neon?

Serverless PostgreSQL with free tier (3 branches, 0.5GB storage). Scales automatically. Best for Vercel deployment.

  1. Create free account at neon.tech
  2. Create a new project: Name: portfolio, Region: closest to you
  3. Go to Dashboard → Connection Details
  4. Copy the Connection string (looks like: postgresql://user:password@ep-xxx-xxx-123456.us-east-2.aws.neon.tech/portfolio?sslmode=require)
💡
Connection String Format

Add ?sslmode=require at the end for secure connection

Backend Setup

Install, configure, and initialize the backend

Navigate to Backend
cd backend
Install Dependencies
npm install
Configure Environment Variables

Create .env file from the example:

cp .env.example .env

Edit .env with your credentials:

DATABASE_URL Your PostgreSQL connection string
JWT_SECRET Random 32+ character string for auth tokens
CLOUDINARY_CLOUD_NAME From Cloudinary Dashboard → Settings
CLOUDINARY_API_KEY From Cloudinary Dashboard → Settings → API Keys
CLOUDINARY_API_SECRET From Cloudinary Dashboard → Settings → API Keys
Initialize Database
# Push schema to database
npm run db:push

# Generate Prisma client
npm run db:generate
Seed Database (Optional)

Creates sample data and admin user:

npm run db:seed
🔑
Default Admin Credentials

Email: admin@rovicmejia.com
Password: admin123

Start Development Server
npm run dev

Backend available at: http://localhost:3000

Frontend Integration

API-connected static pages

The frontend is already connected to the backend API. The inquiry forms work automatically.

4.1 Include API Integration Script

Add to HTML pages for dynamic content loading:

<script src="api-integration.js"></script>

4.2 How It Works

📤 Form Submission

Forms POST to /api/inquiries → saved to database

📧 Fallback

If API unavailable, falls back to mailto: email

📥 Data Loading

Use PortfolioAPI.loadPortfolioItems() for dynamic content

4.3 Load Dynamic Content

// Load portfolio items
PortfolioAPI.loadPortfolioItems('.featured-grid', { featured: true, limit: 4 });

// Load services
PortfolioAPI.loadServices('.services-grid');

// Load blog posts
PortfolioAPI.loadBlogPosts('.blog-grid', { published: true });

4.4 Manual API Calls

// Fetch data
fetch('/api/portfolio')
  .then(res => res.json())
  .then(data => console.log(data));

// Submit inquiry
fetch('/api/inquiries', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com',
    message: 'Hello!'
  })
});

Deploy to Vercel

Make your portfolio live

Install Vercel CLI
npm install -g vercel
Deploy
# Navigate to project root
cd portfolio-new

# Deploy
vercel

# For production
vercel --prod
Configure Environment Variables

In Vercel Dashboard → Settings → Environment Variables:

DATABASE_URL PostgreSQL connection string
JWT_SECRET Authentication secret key
CLOUDINARY_CLOUD_NAME Your cloud name
CLOUDINARY_API_KEY API key
CLOUDINARY_API_SECRET API secret
Redeploy

After setting environment variables, trigger a new deployment from the Deployments tab.

🎉
Deployment Complete!

Website: https://your-project.vercel.app
Admin: /admin

Troubleshooting

Common issues and solutions

Database Connection Error

Check that your DATABASE_URL is correct and the database is accessible. For cloud DBs, ensure your IP is whitelisted.

Prisma Client Not Generated

Run npm run db:generate after changing the schema.

API Returns 500

Check server logs. Usually caused by missing environment variables or database issues.

Images Not Uploading

Verify Cloudinary credentials in environment variables.
